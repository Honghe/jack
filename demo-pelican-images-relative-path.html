Demo Pelican images relative path

Tue 23 February 2021
By Jack
Category: CS

Show image in directory relative to current article file path.

├── content
│   └── test_images_relative_path
│       ├── images
│       │   └── smile.png
│       └── test_images_relative_path.md

How to use smile.png image, we have to add {static} prefix.

![]({static}./images/smile.png)

Rendered html output:

<img alt="" src="/test_images_relative_path/images/smile.png">

We can see the image now.

But how can we do this without {static} prefix? Bause the {static} prefix is not friendly with other Markdown editor such Typora.
